◆ rfm75_deferred_interrupt()

uint8_t rfm75_deferred_interrupt ( )

Handle RFM75 IRQ, posting to the registered RX and TX callbacks as needed.

This function needs to be called every time that the RFM75 IRQ is asserted, preferably as soon as possible. However, for performance reasons it's important that this not be called from inside the interrupt service routine itself. The ISR sets a flag called f_rfm75_interrupt, which signals to the main program that this function needs to be called. While this deferred interrupt is pending, the radio will have limited to no background functionality (depending on whether we are in PTX or PRX mode).

This function will, as needed, clear the interrupt vector on the RFM75, and clear the interrupt flag that was set in this driver's ISR.

void rfm75_tx ( uint16_t  addr,
uint8_t  noack,
uint8_t *  data,
uint8_t  len 
)

Transmit an RFM75 message to a given address, or RFM75_BROADCAST_ADDR.

Parameters
addrThe destination address, or RFM75_BROADCAST_ADDR.
noackDisable acknowledgments. This is only valid when addr is a unicast destination, because broadcast messages can't be acknowledged anyway.
dataA pointer to the buffer containing the data to transmit.
lenThe length of the data buffer.

Note that it's important for len to be the same as RFM75_PAYLOAD_SIZE, or else strange things may happen.
